Redis笔记
redis nosql数据库
发布日期:   2018-10-29
阅读次数:  

redis

内存级别的操作，存取快
支持String、集合等数据类型
支持事务
数据库、缓存和消息中间件

redis 持久化有几种方式

rdb（redis database）和aof（append of file） 第一个时快照文件写入磁盘的 第二个时记录每条操作日志

使用场景

配合关系型数据库做高速缓存
缓存高频次访问的数据，降低数据库io
分布式架构，做session共享

redis 和 memecache 有什么区别

redis能持久化
redis支持类型多

redis是单线程

因为Redis是基于内存的操作，CPU不是Redis的瓶颈，Redis的瓶颈最有可能是机器内存的大小或者网络带宽。
既然单线程容易实现，而且CPU不会成为瓶颈，那就顺理成章地采用单线程的方案了

缓存击穿

使用锁或者布隆过滤器解决

缓存雪崩

一段时间内大量key失效 从而去数据库查询
数据预热：在发生并发前先更新缓存 设置不同的失效时间

缓存更新

定时更新 或者请求时发现失效再更新

Redis支持的Java客户端都有哪些

Redisson,Jedis，lettuce

jedis 和 redisson 有哪些区别

Jedis 只是简单的封装了 Redis 的API库，可以看作是Redis客户端，它的方法和Redis 的命令很类似。Redisson 不仅封装了 redis ，还封装了对更多数据结构的支持，以及锁等功能

数据库数据一致性问题

设置过期时间，以DB写为准 缓存写尽最大努力
尽量不缓存那些对一致性要求高的数据

淘汰策略

不删除策略 对所有key删除最少使用的 对所有key随即删除一部分 对过期key删除最少使用的 对过期key随即删除一部分 对过期key随即删除剩余时间少的

性能问题

aof持久化时 数据过多 启动慢 消耗CPU和内存 会出现服务暂停

一张自增表里面总共有 7 条数据，删除了最后 2 条数据，重启 mysql 数据库，又插入了一条数据，此时 id 是几

6 默认时innodb 最大id时存到内存的 重启后内存清除 最大id变为5 新插入是6
如果是myisam 最大id存到文件中 重启后id还是7 新插入是8

select version()
